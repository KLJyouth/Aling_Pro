API介绍

AlingAi Pro API提供了一套全面的人工智能接口，让您能够轻松地将AI能力集成到您的应用程序、服务或产品中。我们的API采用RESTful设计原则，为您提供简单直观的集成体验。

核心特性

  • 高性能 REST API

    所有API端点都经过优化，提供毫秒级响应时间和99.99%的可用性

  • 全球分布式架构

    遍布全球的服务器集群确保低延迟访问和数据冗余

  • 企业级安全

    所有API通信采用TLS 1.3加密，支持API密钥和OAuth 2.0认证

  • 灵活的定价模型

    按需付费或企业级套餐，根据您的使用量自动扩展

  • 详细文档与SDK

    提供多种语言的SDK和详尽的API文档，加速开发过程

基本API端点

https://api.alingai.pro/v1/

所有API请求都以此基础URL开头，后接特定的资源路径

开始使用

创建API密钥

  1. 登录您的AlingAi Pro开发者控制台
  2. 进入"API密钥"部分
  3. 点击"创建新密钥"按钮
  4. 为您的API密钥命名并设置权限范围
  5. 保存生成的API密钥（注意：密钥只会显示一次）

请求示例

GET
/v1/ai/models
curl -X GET "https://api.alingai.pro/v1/ai/models" \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json"

认证

AlingAi Pro API支持两种主要的认证方式：API密钥认证和OAuth 2.0认证。对于大多数开发者而言，API密钥认证足够简单且安全。对于需要更高级访问控制的企业用户，我们推荐使用OAuth 2.0认证。

API密钥认证

API密钥是访问AlingAi Pro API最简单的方式。每个密钥都与您的账户关联，并具有指定的权限范围和限制。

HTTP头部认证方式

Authorization: Bearer YOUR_API_KEY

将您的API密钥添加到请求头中。这是推荐的方法，因为它可以避免密钥暴露在URL中。

示例请求 
curl -X POST "https://api.alingai.pro/v1/ai/text/generate" \
     -H "Authorization: Bearer ak-xxxxxxxxxxxxxxxxxxx" \
     -H "Content-Type: application/json" \
     -d '{
       "prompt": "使用人工智能解决气候变化问题",
       "max_tokens": 500,
       "temperature": 0.7
     }'

查询参数认证方式

https://api.alingai.pro/v1/ai/models?api_key=YOUR_API_KEY

不推荐 - 此方法仅在不支持自定义HTTP头部的环境中使用，因为它可能导致API密钥暴露在服务器日志中。

示例请求 
curl -X GET "https://api.alingai.pro/v1/ai/models?api_key=ak-xxxxxxxxxxxxxxxxxxx" \
     -H "Content-Type: application/json"

错误处理

AlingAi Pro API使用HTTP状态码表示请求的结果。除了标准HTTP状态码外，错误响应还包含一个JSON对象，提供有关错误的更多详细信息。

HTTP状态码

状态码 描述
200 OK 请求成功处理并返回结果
400 Bad Request 请求参数无效或缺失必要参数
401 Unauthorized 认证失败或API密钥无效
403 Forbidden 权限不足以执行请求的操作
404 Not Found 请求的资源不存在
429 Too Many Requests 超出API调用频率限制
500 Internal Server Error 服务器内部错误

错误响应格式

当发生错误时，响应主体将包含一个JSON对象，提供有关错误的更详细信息：

错误响应示例 
{
  "error": {
    "code": "invalid_request",
    "message": "请求参数无效: 'temperature' 必须在0.0-1.0范围内",
    "param": "temperature",
    "type": "validation_error",
    "request_id": "req_7PzhMeyGj92cBmzV"
  }
}

文本生成API

文本生成API是AlingAi Pro最核心的功能之一，它允许您根据提示生成自然、连贯且高质量的文本内容。您可以控制生成过程的多个方面，包括输出长度、创意程度和特定主题。

端点

POST
/v1/ai/text/generate
https://api.alingai.pro/v1/ai/text/generate

请求参数

参数名称 类型 必填 描述
prompt string 是 用于生成文本的提示
max_tokens integer 否 生成的最大标记数，默认为100，最大为4000
temperature float 否 控制生成文本的随机性，0.0-1.0之间，默认为0.7
model string 否 使用的模型ID，默认为"aling-text-v1"
top_p float 否 核采样参数，0.0-1.0之间，默认为1.0
stop string[] 否 生成遇到这些字符串时停止

请求示例

cURL 
curl -X POST "https://api.alingai.pro/v1/ai/text/generate" \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{
       "prompt": "描述一种未来智能家居系统如何改善人们的生活质量",
       "max_tokens": 500,
       "temperature": 0.8,
       "model": "aling-text-v1"
     }'

高级功能与示例

引导式生成

通过提供明确的指令和示例，您可以引导模型生成特定格式或风格的文本。 

{
  "prompt": "将以下概念转换为技术博客文章的开头段落：\n\n概念：量子计算如何改变网络安全格局",
  "max_tokens": 250,
  "temperature": 0.7
}

多语言支持

API支持多种语言的文本生成，并且可以在不同语言之间进行翻译。 

{
  "prompt": "把以下中文内容翻译成英文：\n\n人工智能技术正在改变我们工作和生活的方式。",
  "max_tokens": 100,
  "temperature": 0.3
}

流式响应

对于需要实时文本生成的应用程序，API支持流式响应，可以在生成过程中逐步返回结果。

只需在端点URL中添加/stream后缀：

https://api.alingai.pro/v1/ai/text/generate/stream 
curl -X POST "https://api.alingai.pro/v1/ai/text/generate/stream" \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{
       "prompt": "分析人工智能在医疗领域的应用前景",
       "max_tokens": 300
     }'

图像生成API

图像生成API允许您通过文本描述生成高质量的图像。这个API可以用于创建各种类型的图像内容，包括产品渲染、场景可视化、概念艺术和创意插图。

端点

POST
/v1/ai/image/generate
https://api.alingai.pro/v1/ai/image/generate

请求参数

参数名称 类型 必填 描述
prompt string 是 要生成图像的文本描述，最大4000个字符
n integer 否 要生成的图像数量，默认为1，最大为4
size string 否 图像尺寸，可选值："256x256", "512x512", "1024x1024", "1024x1792", "1792x1024"，默认为"1024x1024"
style string 否 图像风格，可选值："natural", "vivid", "artistic", "quantum"，默认为"natural"
response_format string 否 响应格式，可选值："url", "b64_json"，默认为"url"
model string 否 使用的模型，默认为"aling-image-v1"

请求示例

cURL 
curl -X POST "https://api.alingai.pro/v1/ai/image/generate" \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{
       "prompt": "一座未来主义风格的城市夜景，霓虹灯光照亮高耸的量子计算建筑，光束在建筑物之间交错，玻璃幕墙反射着紫色和蓝色的光芒",
       "n": 1,
       "size": "1024x1024",
       "style": "quantum"
     }'

响应示例

{
  "created": 1716310846,
  "data": [
    {
      "url": "https://image.alingai.pro/images/generated/a7b9c5d3e2f1g8h6i4j2k0l9m7n5o3p1q8r6s4t2u0v9w7x5y3z1.png"
    }
  ]
}

当 response_format 设置为 b64_json 时，响应会包含Base64编码的图像数据： 

{
  "created": 1716310846,
  "data": [
    {
      "b64_json": "iVBORw0KGgoAAAANSUhEUgAAA..."
    }
  ]
}

聊天对话API

聊天对话API是一个功能强大的接口，支持创建持续的对话，允许AI助手通过上下文记忆与用户进行自然、连贯的交流。这个API特别适合构建聊天机器人、虚拟助手和客户服务应用。

端点

POST
/v1/ai/chat/completions
https://api.alingai.pro/v1/ai/chat/completions

请求参数

参数名称 类型 必填 描述
messages array 是 消息数组，包含对话历史及新的用户消息
model string 否 使用的模型ID，默认为"aling-chat-v1"
temperature float 否 控制生成文本的随机性，0.0-1.0之间，默认为0.7
max_tokens integer 否 生成的最大标记数，默认为1000，最大为8000
stream boolean 否 是否启用流式响应，默认为false
tools array 否 可供模型调用的工具列表

消息格式

messages 数组中的每个消息对象必须包含以下字段：

字段 类型 描述
role string 消息发送者角色，可选值："system", "user", "assistant", "tool"
content string 消息内容
name string 可选，发送者名称，仅在role为"tool"时需要
tool_calls array 可选，工具调用信息，仅在role为"assistant"且调用工具时使用

请求示例

cURL 
curl -X POST "https://api.alingai.pro/v1/ai/chat/completions" \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{
       "model": "aling-chat-v1",
       "messages": [
         {
           "role": "system",

流式响应

通过启用 stream 参数，可以获得实时的响应流，适用于打字机效果的界面实现： 

curl -X POST "https://api.alingai.pro/v1/ai/chat/completions" \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{
       "model": "aling-chat-v1",
       "messages": [{"role": "user", "content": "讲一个简短的科幻故事"}],
       "stream": true
     }'

服务器将以Server-Sent Events (SSE) 格式返回流式响应，每次发送部分内容。

  • • 对于长对话，考虑使用摘要技术或删除较早的、不太相关的消息
  • • 使用适当的temperature参数调整回答的创造性（较低值更确定性，较高值更创造性）
  • • 在敏感场景中，通过系统提示和适当的模型参数控制输出内容

语音识别API

语音识别API将音频转换为文本，支持多种语言和特殊领域词汇。该API适用于语音助手、会议记录、字幕生成等多种应用场景，提供高准确度的实时或批量音频转录。

端点

POST
/v1/ai/speech/transcriptions
https://api.alingai.pro/v1/ai/speech/transcriptions

请求参数

此API使用 multipart/form-data 格式，允许直接上传音频文件：

参数名称 类型 必填 描述
file file 是 要转录的音频文件，格式支持mp3, wav, m4a, flac, webm，最大大小为25MB
model string 否 使用的模型ID，默认为"aling-speech-v1"
language string 否 音频语言的ISO-639-1代码（如"zh"表示中文，"en"表示英文）。如未指定，API将自动检测语言
response_format string 否 响应格式，可选值："json", "text", "srt", "vtt"，默认为"json"
temperature float 否 采样温度，0.0-1.0之间，较低值使音频转文本更加确定，默认为0.2
timestamp_granularities array 否 包含时间戳信息的粒度，可选值："word"（词级别）、"segment"（段落级别），默认为空

请求示例

cURL 
curl -X POST "https://api.alingai.pro/v1/ai/speech/transcriptions" \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: multipart/form-data" \
     -F file=@"recording.mp3" \
     -F model="aling-speech-v1" \
     -F language="zh" \
     -F timestamp_granularities="[\"segment\"]"
