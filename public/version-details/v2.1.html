多租户架构

首次引入多租户概念的重要版本，为不同用户群体提供隔离的工作环境。支持数据隔离、用户分组、 资源配额管理等基础多租户功能，为后续企业级功能奠定架构基础。

数据隔离 用户分组 资源配额 独立配置

多租户核心特性

v2.1版本引入的关键多租户功能

数据隔离

通过数据库级别的逻辑隔离，确保不同租户的数据完全分离，互不干扰，保障数据安全和隐私。

Schema级隔离
租户ID标识
查询自动过滤

用户分组管理

支持将用户分配到不同租户，实现用户群体的逻辑分组，每个租户可以独立管理自己的用户。

租户用户映射
角色权限继承
跨租户访问控制

资源配额管理

为每个租户设置独立的资源使用配额，包括存储空间、API调用次数、并发用户数等资源限制。

存储配额限制
API调用限制
并发用户控制

独立配置空间

每个租户拥有独立的配置空间，可以自定义安全策略、界面主题、功能开关等个性化设置。

安全策略配置
界面主题定制
功能模块开关

租户生命周期

提供完整的租户生命周期管理，包括租户创建、配置、监控、暂停、删除等全流程操作。

动态创建租户
租户状态管理
资源回收机制

租户监控统计

实时监控各租户的资源使用情况、用户活跃度、API调用统计等关键指标，便于管理和优化。

资源使用监控
用户活跃统计
性能指标分析

多租户升级命令 

# 1. 备份现有数据
./alingai-backup.sh --full-backup

# 2. 升级数据库Schema
./alingai-db-upgrade.sh --multi-tenant-schema

# 3. 升级应用到v2.1
./alingai-upgrade.sh --version=v2.1 --multi-tenant

# 4. 初始化默认租户
./alingai-tenant.sh --create-default --name="Default"

# 5. 迁移现有用户到默认租户
./alingai-migrate-users.sh --to-tenant="Default"

# 6. 验证多租户功能
./alingai-verify.sh --multi-tenant-test
